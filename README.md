# VSLI projects
a git for all VSLI module work


## Using GIT
step 1: git pull (updates your files to the ones on github do BEFORE working)

step 2: git add * (selects ALL files to be added, will only change files that have been changed)

step 3: git commit -m "message" (will associate message "message" with the changelist from the add command)

step 4: git push (pushes changes to github)

## Special Purpose FPU
all files are dumped in one folder.

.out files (testbenches) to be saved as tb_xxx_xxx.out (e.g. tb_double_adder.tb)

.vcd files (waveforms) to be saved as gtk_xxx_xxx.vcd (e.g gtk_double-adder.tb)

**NOTE** .vcd files are too big for github and wont be saved to github. run locally to get them for testing
